# 06 : I/O

The TI-BASIC I/O surface is partitioned by which screen is active. The home screen has its own command set; the graph screen has another (covered in [07-graphics.md](07-graphics.md)). Programs typically pick one and stay there. Switching between screens mid-program is permitted but flickers and costs tokens.

## Home screen geometry

Monochrome (TI-83 / 83+ / 84+ / 84+SE):
- 8 rows by 16 columns.
- Single fixed font (5x7 pixels per cell).
- Coordinates are 1-indexed: `(1,1)` is top-left, `(8,16)` is bottom-right.

84+CSE: 10 rows by 26 columns.

84+CE: 10 rows by 26 columns, color (foreground / background palette is selectable; details in 84+CE-only commands).

## Home-screen commands

### `ClrHome`

Clears the home screen. Conventional at program start and end.

### `Disp`

Prints one or more arguments on a new line, scrolling the previous content up.

```
:Disp "HELLO"
:Disp A, B, "END"
```

- Arguments are comma-separated. Each prints on its own line **unless mixed with** other constraints (multi-arg `Disp` does emit multiple lines).
- A bare `Disp` (no args) advances to the next line.
- Strings without trailing `"` are auto-closed (`Disp "HI`).
- `Disp` of a list / matrix prints in the standard list / matrix display form.

### `Output(`

Prints at a specific cell, **without scrolling**:

```
:Output(row, col, value)
```

- Coordinates are 1-indexed. Row 1, column 1 is top-left.
- `value` may be a number, string, or expression. Multi-token strings wrap around the screen.
- `Output(` does not clear the cell first; trailing characters from prior output remain. Pad with trailing spaces in the string if you need a clean overwrite.
- 84+CSE / CE add color foreground/background variants; on monochrome the visual is fixed.

### `Pause`

Halts the program until the user presses `ENTER`.

```
:Pause
:Pause "MESSAGE"
:Pause value
```

- Bare `Pause` waits without printing anything; the home screen retains its current contents.
- `Pause "MESSAGE"` displays the message (subject to the same scrolling as `Disp`) before waiting.
- `Pause` displays a small busy / pause indicator in the upper-right.

### `Menu(`

Pops a menu and dispatches to one of up to seven labels:

```
:Menu("TITLE", "OPT1", L1, "OPT2", L2[, "OPT3", L3, ...])
```

- Each option pairs a label string with a `Lbl` target.
- Selection: user presses `1`-`7` or arrows + `ENTER`.
- Dispatch is via `Goto`, so all the leak / structure caveats for `Goto` apply (see [05-control-flow.md](05-control-flow.md)).
- The title appears at the top; up to 7 options fit on monochrome.
- Pressing `[ON]` cancels and raises `ERR:BREAK`.

## User input

### `Input`

Two forms:

```
:Input variable                    // bare prompt "?"
:Input "PROMPT", variable          // with a custom prompt
:Input "PROMPT:", Str1             // string-valued
:Input                             // graphics-mode crosshair (graph screen)
```

- The variable type determines the parser: storing into a real reads a number, into a string reads characters until ENTER.
- A string `Input` accepts unquoted text. The user pressing `"` types a literal `"` into the destination string.
- A bare `Input` (no args) on the graph screen presents a crosshair the user moves with arrow keys; storing on `ENTER` to `X`, `Y`. Useful for picking coordinates.
- `Input` echoes the prompt and reads a line ending in `ENTER`. No partial-line interaction.

### `Prompt`

Like `Input`, but takes one or more variables and prompts each in sequence with the variable name:

```
:Prompt A, B, Str1
```

Displays `A=?`, then `B=?`, then `Str1=?`. Smaller than three `Input` lines when a literal prompt isn't needed.

### `getKey`

The single asynchronous input primitive. Returns the key code of whichever key is currently pressed, or 0 if none. **Does not block.** Use in a loop to poll.

```
:Repeat Ans=105
:getKey
:If Ans:Disp Ans
:End
```

Key codes are `row * 10 + col` of the calculator keyboard layout. The `[ON]` key has no code (it always raises `ERR:BREAK` instead).

### Key code map (monochrome 83/83+/84+)

The keypad is laid out in 10 rows. Codes are `(row * 10) + col`; rows count from the top, columns from the left.

|       | col 1 | col 2 | col 3 | col 4 | col 5 |
|-------|-------|-------|-------|-------|-------|
| row 1 |   11  |   12  |   13  |   14  |   15  (`F1` arrow keys)        |
| row 2 |   21  |   22  |   23  |   24  |   --  (`UP`=25 in this row)   |
| row 3 |   31  |   32  |   33  |   --  |   34  (`DOWN`=34 in this row) |
| row 4 |   41  |   42  |   43  |   44  |   45                         |
| row 5 |   51  |   52  |   53  |   54  |   55                         |
| row 6 |   61  |   62  |   63  |   64  |   65                         |
| row 7 |   71  |   72  |   73  |   74  |   75                         |
| row 8 |   81  |   82  |   83  |   84  |   85                         |
| row 9 |   91  |   92  |   93  |   94  |   95                         |
| row 10|   --  |   102 |   103 |   104 |   105 (`ENTER`)              |

The four arrow keys: `UP=25`, `RIGHT=26`, `LEFT=24`, `DOWN=34`. `ENTER=105`. `MODE=45`. `2nd=21`. `ALPHA=31`. `[ON]` returns no code.

If you depend on the precise integer, validate against the live calculator: a small `getKey` echo program is the canonical reference (the calc's view of itself, not the documented map).

## Output formatting and modes

Several mode flags affect numeric display from `Disp`, `Output(`, and the home screen:
- `Normal` / `Sci` / `Eng` : decimal, scientific, engineering notation.
- `Float` / `Fix n` : floating decimal places or a fixed count.
- `Radian` / `Degree` : trig display and input.
- `Func` / `Param` / `Polar` / `Seq` : graphing mode (rarely affects `Disp`, but `getKey` and the graph screen behave differently).
- `Connected` / `Dot` : graph plot style.
- `Sequential` / `Simul` : simul or sequential graphing.
- `Real` / `a+bi` / `re^thetai` : complex number handling.
- `Full` / `Horiz` / `G-T` : split-screen mode.

Set with their token: `:Float`, `:Fix 2`, `:Degree`, `:Real`. Reset to user-preferred state at end-of-program if your program changed them.

## ` ` character is meaningful in strings

Spaces inside `"..."` are tokens and they print. `:Disp "HELLO   WORLD"` produces three spaces between words. There is no width-trim or alignment built in; pad manually.

## Program-to-program I/O

There is no stdin/stdout. Programs communicate by sharing variables (real, list, etc.). `prgm` calls inherit the entire variable namespace. Conventionally:
- Caller writes "argument" registers (`Z`, `theta`).
- Callee reads them, writes "result" registers (`Ans` or another fixed real).
- Caller reads results.

This is informal; document the convention in a header string-comment.

## Linking I/O (calculator-to-calculator / calc-to-host)

The `Send(` / `Get(` family transmits variables over the calculator's link port. These are essentially blobs over the proprietary DBUS / silver-link protocol; details belong to the firmware and tooling layer (see `docs/hackspire/03-os-and-comms.md` for the wire format and `firmware/c` for the project's host side).

In TI-BASIC:
- `Send(varlist)` transmits.
- `Get(var)` blocks until the matching variable arrives.
- `OpenLib(libname)` / `ExecLib` (84+ only) load a Flash app library; primary user is `usb8x` for USB.

A program using these is portable only across calculators that have the same link primitive. The TI-83 uses `Send(9prgmNAME)` for a different purpose (asm program launch); same token, different shape, not portable to TI-83+.
